# creative-interview — Stage 2〜3の実行手順

目的: 解析で埋まらない情報のうち、**表現判断に影響するものだけ**をユーザーに質問する。
方法論の正は `docs/creative/CREATIVE_METHODOLOGY.md` 2.2（仮説検証型Interview）。

## Stage 2: Information Gap Analysis

1. project-analysisの情報ギャップ一覧を見る
2. 各[未確認]を仕分ける:
   - **解析で埋まる** → 追加調査して埋める（質問しない）
   - **表現判断に影響しない** → Open Questionsに記録して先へ進む
   - **表現判断に影響し、解析で埋まらない** → Interview候補
3. Interview候補がゼロなら、Stage 3をスキップして理由を記録する（**Interviewは情報不足時だけ実行する**）

## Stage 3: Creative Interview

### 規則

1. **既に回答済み・資料にある情報を再質問しない**（信頼を削る）
2. **開いた質問より仮説検証** — 「こだわりは？」ではなく「◯◯が一番大切な場所に見えました。合っていますか？」。解析から仮説を立て、確認の形で聞く
3. **答えやすい順** — 事実確認 → 好き嫌い（避けたい印象）→ 核心（なぜこの商売か）。核心をいきなり聞かない
4. **一度に投げる質問は7問以下を目安** — 重要度順。足りなければ回答を見てから追加する
5. **回答は原文のまま記録する** — 言い回し自体が世界観の証拠・コピーの原料
6. **「避けたい印象」は未確認なら必ず聞く**（QUALITY_STANDARD 1章 [Major]）

### 質問領域メニュー（全部聞くリストではない。未確認のものだけ選ぶ）

起源／最も大切にしている価値／顧客に感じてほしいこと／他社・他店との違い／最も見せたい商品・サービス／空気感（五感で）／最も自信のある点／ブランドを一言で表す言葉／希望するサイト体験／**避けたい印象**／望むスクロール体験／サイトで達成したい成果／顧客にどう紹介されたいか／ブランドを映画に例えると

## 参考サイト希望の翻訳（模倣の回避）

ユーザーが「WEDNESみたいに」「Appleみたいに」等、特定サイトを挙げた場合:

1. そのまま様式を写すことは禁止（憲章6章: 模倣の禁止）と、丁寧に一度だけ伝える
2. **参照を性質に翻訳する質問**をする — 「そのサイトの**何が**そう感じさせましたか？ 速度ですか、静けさですか、没入感ですか、次を見たくなる感じですか？」
3. 得られた性質（例: 「先が気になって最後まで見てしまう」）を、参照名を捨てて設計原理として記録する（例: Curiosity設計の要求）
4. 翻訳に応じてもらえず「同じものを作って」に留まる場合は停止条件（模倣要求）として報告する

## ユーザーが回答できない場合

- 一部が未回答 → 該当項目を[未確認]のままBriefのOpen Questionsへ。設計は確からしさの高い情報の範囲で行い、仮定を明示する
- **ブランドの核（価値・差別化・避けたい印象）がすべて埋まらず、資料からも推定できない** → 設計根拠が作れないため停止し、必要な情報・入手方法の提案とともに報告する
