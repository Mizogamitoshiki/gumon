# Gumon Pattern Catalog — Bloom-L Engine 還元用

作成: 2026-07-11 ／ 愚問（GUMON）v1検証案件で確立した、再利用可能な設計パターン7種。
**組み合わせ（体験そのもの）は案件固有のまま。ここに記録するのは「部品・原則」のみ**（憲章7.8 Originality）。

---

## 1. Cinematic Storytelling（TOPページ・単一pinステージ型）

- **使う条件**: ブランドの世界観そのものを1本の物語として語りたいトップページ。企業・店舗の中核メッセージが明確で、感情曲線を作れるだけの素材（コピー・写真・動画）がある
- **使わない条件**: 情報検索が主目的のページ（メニュー詳細・アクセス等）。素材が薄く感情曲線を支えられない場合
- **Story上の役割**: 起点（出会い）から結末（CTA）までを1本道で運ぶ。物語の要約が1〜2文で言える
- **Motion密度**: 4-5（サイト最高密度。ただし高強度連続≤2・低強度区間≥20%を厳守）
- **必要素材**: 章句コピー・映像/写真複数点・感情曲線の設計文書
- **Mobile方針**: 完全静的な縦積みへ全面代替（pin・T体系は使わない）
- **reduced-motion方針**: 全Motion停止・静的表示
- **QA条件**: Scroll Rhythm（高強度連続≤2）・Transition単調性チェック・削除テスト必須
- **主要コンポーネント**: `GumonScroll.tsx`（単一マスタータイムライン・Lenis統合）
- **他案件への注意点**: T体系・フィルムスクラブ・章句は「TOP限定・下層へ持ち込まない」と最初から明文化すること。素材が薄い案件では無理に適用しない

## 2. Dinner Scene Turn Editorial（単一ステージpin型・活気）

- **使う条件**: 「対面する」「儀式的に選ぶ」信頼形成の核となる詳細ページ。実写が1点以上あり、看板商品を強く見せたい
- **使わない条件**: 実写ゼロのページ。素早い決定を優先すべきページ（Brisk向き）
- **Story上の役割**: 一皿と対面する→選ぶ→席を決める、という儀式的な流れ
- **Motion密度**: 3（quiet分岐: riseLine・y40・波状連鎖）
- **必要素材**: 看板商品の実写1点以上
- **Mobile方針**: 完全静的（gm-stage自体を無効化）
- **reduced-motion方針**: 静的
- **QA条件**: pin-spacer 0確認・375px横スクロール0・CTAボタン常時操作可能性
- **主要コンポーネント**: `DishShowcase.tsx`・`MenuBoard.tsx`(`quiet` prop)
- **他案件への注意点**: 単一ステージpinは「1ページ1箇所まで」。増やすとScroll Rhythmの高強度連続が崩れる

## 3. Lunch Brisk Menu（軽量Editorial・速い決断）

- **使う条件**: 「速く・軽く決めたい」来店動機のページ（ランチ・SEO流入主力等）。実写が乏しくてもコピー中心で成立させたい
- **使わない条件**: じっくり吟味させたいページ（Dinner/Course向き）
- **Story上の役割**: 入口に出会う→迷わず選ぶ→今日の実務を確かめる
- **Motion密度**: 2（brisk分岐: riseLineベースだがquietより移動量小・stagger速い）
- **必要素材**: なし（実写ゼロ前提で設計可能）
- **Mobile方針**: 完全静的
- **reduced-motion方針**: 静的
- **QA条件**: 空ギャラリーの削除テスト（実写がなければ確実に外す）
- **主要コンポーネント**: `MenuBoard.tsx`(`brisk` prop)・DishGallery非描画
- **他案件への注意点**: 「速さ」を煽るコピーの追加は捏造リスクがある。既存コピーの範囲で速度を演出に閉じること

## 4. Course Consultation Editorial（相談誘導・最弱静けさ）

- **使う条件**: 「選ばせる」より「相談させる」ページ（宴会・貸切・カスタムオーダー等、条件が個別化するサービス）
- **使わない条件**: 定型商品を選ぶだけのページ（Lunch/Dinner向き）
- **Story上の役割**: 相談の入口→対応できることを知る→電話で席を整える
- **Motion密度**: 1（consult分岐: fadeQuiet基準・y8px・最弱）
- **必要素材**: 必須なし（実写ゼロ前提で成立させる設計）
- **Mobile方針**: 完全静的
- **reduced-motion方針**: 静的
- **QA条件**: 電話ボタンを演出対象から除外（常時操作可能）・notes内電話番号のtelリンク化・Contact導線は電話CTAより明確に弱く
- **主要コンポーネント**: `MenuBoard.tsx`(`consult` prop)・`renderNote()`ヘルパー
- **他案件への注意点**: 「ご相談ください」を安易な逃げ口上にしない。既存の誠実なコピーがあってこそ成立するパターンであり、コピー自体を新規に薄めて作ってはならない

## 5. Drink Calm Editorial（余韻・consultと強度共有）

- **使う条件**: 主要商品の「余韻・付随」を担うページ。既に確立した最弱パターン（Course）と同じ強度で十分な場合
- **使わない条件**: 独自の強い個性を打ち出す必要があるページ
- **Story上の役割**: 前の体験の余韻→対象を知る→行動へ戻る
- **Motion密度**: 1（Courseとの意図的な強度共有。新規マジックナンバーを作らない判断そのものがパターン）
- **必要素材**: 必須なし
- **Mobile方針**: 完全静的
- **reduced-motion方針**: 静的
- **QA条件**: 「Lunch以下・Course以上」等、強度範囲の指示を受けた場合は下限（最弱側）に寄せて理由記録を簡潔にする
- **主要コンポーネント**: `MenuBoard.tsx`(`calm` prop・`consult || calm`条件で分岐統合)
- **他案件への注意点**: 似た役割の複数ページがある場合、強度値を無理に差別化せず「共有する判断」自体を理由として記録してよい（Motion Has Meaningは「差別化の理由」だけでなく「共有する理由」も要求する）

## 6. About Reading Editorial（本文読解優先・行Reveal静的化）

- **使う条件**: ブランドの由来・思想を語る長文ページ。TOPで一部コピーを先取り済みの場合の重複回避が必要
- **使わない条件**: 短文中心のページ（Utility向き）
- **Story上の役割**: 問いの「本文」。TOPの要約を深掘りする
- **Motion密度**: 1（見出しmask/eyebrow/罫線のみ残し、本文行Revealは静的化）
- **必要素材**: なし（既存コピーの再配置のみで対応可能な場合が多い）
- **Mobile方針**: 完全静的
- **reduced-motion方針**: 静的
- **QA条件**: TOPとの完全一致コピーは削除ではなく「再配置」で反響化することを優先検討（Story Quality: 情報開示の順番に理由）
- **主要コンポーネント**: `InfoSection.tsx`・`PullQuote.tsx`
- **他案件への注意点**: 重複解消は「削除」より先に「再配置（順序変更）」を検討する。コピーの変更・削除が必要な場合は必ず変更候補ドキュメントに分離し、無断で書き換えない

## 7. Utility Information Page（Access/Contact/Recruit・実用最優先）

- **使う条件**: 来店・予約・応募など、訪問者の実務タスク達成が最優先のページ
- **使わない条件**: ブランド体験の演出を主目的とするページ
- **Story上の役割**: 演出を足さない。既存の情報構造そのものが役割を果たす
- **Motion密度**: 0-1（新規演出を追加しない。既存のInfoSection標準Revealのみ）
- **必要素材**: なし
- **Mobile方針**: 1タップ発信・44px以上のタップ領域を最優先
- **reduced-motion方針**: 静的
- **QA条件**: タップ領域44px未満の要素を全数チェック（本パターンで最も再発しやすい不具合）。電話CTAを演出対象から除外
- **主要コンポーネント**: `InfoSection.tsx`・`TelCta.tsx`・`FaqList.tsx`
- **他案件への注意点**: 「Utilityだから何もしなくていい」ではない。タップ領域・電話CTAの常時操作可能性・FAQのキーボード操作性など、地味だが実務に直結する点を毎回チェックする

## 横断的に確立したパターン（7種に共通）

- **Variant分岐の設計原則**: 新しいページ性格を追加する際は、既存分岐（quiet/brisk/consult/calm等）の値を逐語で保持したまま独立した`else if`を追加する。共有ファイルの`git diff`で削除行を確認し、既存分岐が変わっていないことを機械的に照合する
- **電話CTA常時可視の原則**: `gsap.from()`でCTAコンテナ全体を対象にすると、主要ボタンがスクロールトリガー発火まで不可視になる。主ボタンは常に演出対象から除外する（`Array.from(cta.children).filter(c => !c.classList.contains(...))`）
- **タップ領域確保の技法**: padding + 同値のnegative marginで、レイアウトを変えずにタップ領域を拡張する（`.gm-note-tel`・`.gm-info-tel-link`）
- **削除テストの型**: 実写ゼロのプレースホルダギャラリーは、品名・価格・説明が別途全件掲載されていれば「情報損失なし」と判定し削除する
