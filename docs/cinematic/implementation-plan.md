### 1.1 現状の分析（要約 — 詳細: project-analysis.md）

Next.js 14 + GSAP/ScrollTrigger + Lenis による pin 820vh・6ビートのscrub LP。確定デザインシステム（黒/アイボリー/ストーン/朱CTA限定・金不使用）とブランド語彙（問い・火・時間・静けさ）で統一されたコピー、スクロールスクラブの料理フィルム（シグネチャームーブ）、reduced-motion完全分岐、ローカルSEO実装を持つ。**技術品質・ブランド表現の土台は既に高い。**

### 1.2 現状の強みと課題

**強み**: ①スクラブフィルムという固有のシグネチャームーブ ②/aboutに眠る一級のブランドコピー群（「おいしいとは、なにか。」等） ③成熟したモーション基盤（単一rAF・reduced-motion・モバイル分岐） ④実データ主義の信頼設計（実価格・FAQ・「お電話がいちばんありがたい」）

**課題**: ①6ビートが**情報の順序**であって**物語**になっていない（ビート間の意味的接続がフィルム以外にない） ②ほぼ全境界が同型のクロスフェード（QS 6章 [Major]相当） ③ピーク（フィルム）の直後に受け止めの静けさがない（QS 7章 [Blocker]相当: 圧倒→集中が直結） ④ブランド最強のコピーがLPに不在で、店名の意味が伝わらない ⑤S9のletter-spacingアニメーションがtransform/opacity規定に厳密には抵触 ⑥CWV・a11y実測が未実施

### 1.3 Creative Concept（creative-brief.md 6章から）

**「一つの愚問を、スクロールが深めていく。」**
ページ全体を「おいしいとは、なにか。」への問答として構成する。訪問者はスクロールという自分の手で問いを進め、素材・火・時間が順に返事をし、最後の答え（一皿）は画面では出さない — 答えは席にしかないから、予約が物語の結末になる。

### 1.4 設計の要約（詳細: experience-plan.md）

- **感情曲線**: 警戒（比較モード）→ 入場の安心 → 問いへの足止め → 期待 → **圧倒（フィルム・唯一のピーク）** → 受け止めの半拍 → 集中（品書き＝証拠）→ 静寂（一杯）→ 安心（実在）→ 親密な信頼 → 行動 → 余韻。ピーク1・静けさ3区間・低強度≈27%
- **Story（一文）**: 「おいしいとは、なにか」という愚問を手渡された訪問者が、素材・火・時間の返事をスクロールで目撃し、最後の答えだけは席でしか出ないと知って、電話を手に取る物語
- **Scene Map（9 Scene）**: ①静けさの入場（hero）②**問いの帳（新設）**③台所の姿勢（about）④火の返事（film・記憶層）⑤**受け止めの半拍（新設）**⑥お品書き（food）⑦一杯の静けさ（drink）⑧路地の奥（access）⑨答えは、席で。（reserve）— 既存6ビートを全て継承し、新設2 Sceneは既存コピー・既存素材の再配置のみで成立
- **Transition Map（8境界）**: 受け渡す→溶ける→変形（窓が開く）→溶ける（減衰）→受け渡す→溶ける（間をおいて）→変形（地図）→溶ける（暗転からの浮上）。同型連続なし・スクロールロックなし
- **Rhythm Score**: 吸う→深く吸う→浅く吸う→**止める**→吐く→浅く吸う→吐く→吐く→深く吐く。「休符」をトークン（hold-quiet）として明示的に予約
- **Motion Plan**: 19行のMotion一覧表（既存継承12・既存変更3・新規4）＋不採用記録5件（WebGL・SplitTextプラグイン・横スクロール・カウントアップ・新規依存）。理由記録率100%・全行にreduced-motion/モバイル代替・削除条件を定義済み
