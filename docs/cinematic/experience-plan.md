## 1. 記憶層の選定

記憶に残す2箇所（これ以外のSceneは意図的に磨きすぎない — 迷ったら減らす）:

1. **「火の返事」— スクロールスクラブの料理フィルム（Scene 4）**: 訪問者が自分のスクロールで火と調理の一本のテイクを進める体験。[根拠] Brief体験ゴール「記憶: 自分が進めたフィルムの手応え」／Brand Essence文化「酢豚の照りも、麻婆豆腐の痺れも、火の返事です」。既存実装唯一のシグネチャームーブを物語の頂点として意味づけ直す
2. **「問いの帳」— 「おいしいとは、なにか。」の提示（Scene 2）**: 現在 /about にしか無いブランド最強のコピーを、LPの物語の起点に昇格させる。[根拠] Brief体験ゴール「記憶: 『問い続ける台所』という一つの概念」／Concept Statement「一つの愚問を、スクロールが深めていく。」— 問いが立たなければフィルムは「きれいな料理動画」にしかならない

## 3. Story

- 物語の一文: **「おいしいとは、なにか」という愚問を手渡された訪問者が、素材・火・時間の返事をスクロールで目撃し、最後の答えだけは席でしか出ないと知って、電話を手に取る物語。**
- 起点と結末の接続: 起点の問いが結末で「答えは、席で。」として回収される。CTAは販促ではなく問いの結末（Trust Before Action）

開示順と理由:

| 順 | 情報 | なぜこの位置か |
| --- | --- | --- |
| 1 | 店名・業態・静けさの気配（hero） | 比較モードの訪問者に3秒で「中国料理・静謐・本物」を伝える（KPI）。世界観の入口 |
| 2 | 問い「おいしいとは、なにか。」＝店名の意味 | 以降すべてのSceneの意味づけ装置。問いが先に立たないと、フィルムも品書きもただの情報になる |
| 3 | 姿勢（問いを重ね、一皿に答える／素材・火・時間） | 問い→どう問うか、の順。次Sceneの3つの返事の伏線を張る |
| 4 | 火の返事（調理フィルム＝仕事の実物証拠） | 思想を語った直後に実物で証明する（世界観→具体の定石）。食欲は最も正直な説得 |
| 5 | お品書き・実価格 | 価格は仕事の証拠の後（taboo「価格訴求を見出しにしない」／Story定石「価格は信頼の後」）。不安①（価格帯不明）をここで解く |
| 6 | 飲み物（一皿に寄り添う一杯） | ピーク後の受け止め区間を兼ねる。食体験の想像を広げ、滞在を深める |
| 7 | 場所・営業時間・地図 | 「行きたい」が立ち上がった後にだけ意味を持つ実用情報 |
| 8 | 予約（電話・Web副次）＋答えの不在の明示 | 物語の結末。信頼が積み上がった地点にのみCTAを置く |

- 捨てる判断: LP上に新規情報ブロックは足さない。採用情報・FAQ・こだわり詳細は既存の詳細ページ（/recruit /contact /about）が受け持ち、LPは物語に専念する（物語の節に割り当てられない情報はページから外す）

- [x] 高強度の連続 ≤ 2（高はS4の1つのみ）
- [x] 低強度区間 ≥ 20%目安（S2≈8% + S5≈4% + S7≈10% + S9末尾≈5% ≈ 27%）
- [x] インパクト直後に「吐く」がある（S4→S5）
- [x] 冒頭=約束（S1は世界観の予感のみ。フィルムも品書きも見せない）/ 末尾=余韻→行動（ロゴの反響→電話番号→静かな©）
- 単調検査: 呼吸列「吸う→深く吸う→浅く吸う→止める→吐く→浅く吸う→吐く→吐く→深く吐く」— 前半は一方向の蓄積（参道の圧縮）、後半は減衰しながらの着地。機械的反復なし
- テンポ配分（スクロール距離の方針）: 吸気区間（S1〜S3）は現行より詰めすぎない、S4は現行のフィルム章の距離を維持（4.5/9.6≒47%は「読む」より「進める」区間として妥当）、S5の半拍とS6→S7の間は**新たに距離を割く**（急がないことのコスト をスクロール距離で払う）
