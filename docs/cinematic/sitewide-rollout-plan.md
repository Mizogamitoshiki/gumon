## 2. 全体情報設計

- **導線（TOP→下層）**: 既存のScene内リンクを正とする — S3→/about、S6→/menu/{lunch,dinner,course}、S7→/menu/drink、S8→/access、S9→/contact・Web予約・Instagram。ヘッダー/ハンバーガーは全ページ共通の直行路
- **下層→予約**: 全ページ末尾に TelCta（電話主役・「お電話がいちばんありがたい」）を維持。menu系はボード直下、Utility系は最上部にも電話（contactは電話カードがhero直下 ✓）
- **ナビの役割**: TOP=物語への入口（data-goスクロール＋ページ遷移）／下層=DetailNav（aria-currentで現在地・戻る手段）。没入と実用の二層構造を維持
- **CTA優先順位**: 全ページ共通で ①電話（朱・唯一の朱使用） ②Web予約（副次・ホットペッパー） ③回遊リンク（ストーン）。順位を入れ替えない
- **SEOとブランド両立**: 下層はSSR静的＋metadata/JSON-LD（Restaurant/Breadcrumb/FAQ）が既に完備。演出はJSの上乗せでSEOに影響しない構造を維持。ローカルキーワード（貝塚〜）は下層タイトルが担い、TOPは物語に専念
- **遷移時に維持する共通体験**: 確定パレット・Noto Serif/Sans・進捗ヘアライン（ScrollProgress）・「問い」の語彙・reduced-motion方針・44pxタップ・静かな速度（ease-out・8〜40px）
- **TOP固有と再利用の切り分け**: 下記3章

### 最初の1ページ — 候補比較

| | **案1: /menu/dinner（推奨）** | 案2: /about |
| --- | --- | --- |
| 物語上の位置 | S6「品書き」→S9「答えは、席で。」の間を埋める“席へ向かう一歩手前” | S2/S3の問いの「本文」 |
| 事業効果 | 予約KPIに直結（回遊先1位・看板と実価格の証拠） | ブランド深化（転換は間接的） |
| 横展開 | **Editorialの型がそのままmenu残り3ページへ**（計4ページ分の効率） | 型の再利用先が少ない |
| 前提条件 | 実写があると理想（無くても文字中心で成立可） | **コピー重複整理の承認が必要**（TOPがS2で同文を先取りした影響） |
| リスク | 素材待ちで見た目が暫定 | 承認往復で停滞しやすい |
| 代表Scene案 | **「看板の品」ギャラリー節**（麻婆豆腐=「火の返事」の実物を、陰影の額縁で1枚主役に。実写1枚が既にある） | 由来の物語節 |

**推奨: /menu/dinner。** 理由: ①TOPが作った食欲と信頼をそのまま予約へ運ぶページで、事業効果が最も直接的 ②ここで確立するEditorialの型（MenuHero＋fade-quiet化したギャラリー/ボード）がmenu残り3ページへ機械的に展開でき、投資効率が最大 ③/aboutはTOPとのコピー重複整理という承認依存の課題があり、型が固まってからの方が判断材料が揃う。
