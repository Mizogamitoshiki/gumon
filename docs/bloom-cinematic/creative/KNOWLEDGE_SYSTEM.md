# KNOWLEDGE_SYSTEM — Bloom-L Knowledge Bible

本書は、Bloom-L Creative Directorが判断に迷ったとき必ず参照する知識体系である。
[CREATIVE_DIRECTOR_ENGINE.md](./CREATIVE_DIRECTOR_ENGINE.md)（以下CDE）が「どう考えるか」のOSであるのに対し、本書は思考の各段階で引かれる百科事典である。

## 本書の位置づけ

- 文書の優先順位における位置: **第4位（Phaseの設計書）**。上位は [PROJECT_CHARTER.md](../PROJECT_CHARTER.md) → [QUALITY_STANDARD.md](../QUALITY_STANDARD.md) → [CLAUDE.md](../BLOOM_RULES.md)
- 本書はCDEの思考フロー（CDE 2章）の**中で**使われる。本書がCDEの思考順序・意思決定を上書きすることはない
- 上位文書と矛盾を発見した場合は、機械的に適用せず停止して報告する（憲章冒頭の規定）

## 知識の三つの掟 — 本書全体に適用される最上位ルール

**掟1: 知識は一般解であり、出発点である。**
本書のすべての記述は「そのトーン・感情・技法の一般的な性質」である。一般解は、Brand Documentの固有記述に**常に敗北する**（CDE 5.4 ステレオタイプ上書き規則）。本書を引用して「北欧だからこうする」と判断した瞬間、それは業種ステレオタイプの直行採用（CDE 10章-2）と同罪である。

**掟2: 知識の引用は理由にならない。**
採用の理由は、whyChain（CDE 9.5）がBrand Document・感情曲線・訪問者理解のいずれかに着地することでのみ成立する。本書は「その判断が効く仕組み」を説明する道具であって、判断の根拠そのものではない。

**掟3: 禁忌は交渉しない。**
各知識エントリの「禁忌」に1つでも該当したら、その知識は使わない。禁忌は演出効果・要望・納期と交換できない。

## 知識エントリの共通構造

各エントリは原則として次の要素で書かれている。

- **本質** — その知識の一言での核
- **効果** — 訪問者の中で何が起きるか
- **採用条件** — どんな条件が揃ったら使ってよいか
- **禁忌** — 使ってはいけない条件
- **注意** — 強度・実装・性能・アクセシビリティ上の留意点

# 1. ブランド知識 — Brand Tone Library

トーンは、ブランドをCDE 5.2の5軸（速度・重量・温度・時間・距離）に置くための語彙である。

**トーンの合成規則**
- 実案件のブランドは単一トーンでは表せない。**主トーン1＋従トーン1〜2**の合成として記述する（例: 和×モダン、ナチュラル×高級）
- 合成で軸が矛盾する場合（例: 伝統×革新）、どちらを取るかはBrand Documentの固有記述で決める。矛盾こそがそのブランドの個性の在り処であることが多い
- 合成結果が本書のどのトーンにも似ていないなら、それは成功の兆候である

### 1.1 高級

- **本質**: 選び抜かれていること。ノイズの不在。時間と手間が費やされた痕跡
- **5軸の典型**: 静・重・伝統寄り・威厳寄り（温度は「何によって高級か」で決まる）
- **表現の一般解**: 大きな余白、少ない語数、遅く小さなMotion、低彩度、絞られた光
- **ステレオタイプの罠**: 「黒×金×明朝体」の自動適用（QUALITY_STANDARD 1章 不合格例）。高級=冷たい・近寄りがたい、の思い込み
- **上書きの起点になる問い**: このブランドは**何によって**高級なのか — 素材か、技術か、時間か、希少性か、人か。答えが変われば表現は全部変わる

### 1.2 ラグジュアリー

- **本質**: 感覚の飽和。日常からの離脱。「満たされる」体験。高級が引き算なら、ラグジュアリーは満たす美学
- **5軸の典型**: 静〜中・重・威厳寄り。感覚性が強い
- **表現の一般解**: 豊かな質感の大きな写真、深い色、素材へのクローズアップ（カメラの寄り）、ゆったりした没入的テンポ
- **罠**: ゴージャス素材の羅列は飽和ではなく下品化する。テンポを上げると一気に安くなる
- **問い**: 訪問者の五感のうち、どの感覚を満たすブランドか（味・肌触り・香り・音・眺め）

### 1.3 カジュアル

- **本質**: 敷居の低さ。等身大。「あなたのままで来ていい」という許し
- **5軸の典型**: 動・軽・温・親密
- **表現の一般解**: 明るい色、丸みのある形、速く弾むMotion、口語のコピー、人の表情
- **罠**: カジュアル=安っぽい、ではない。作りの雑さはカジュアルではなく手抜きである。ふざけすぎは信頼（Trust Before Action）を毀損する
- **問い**: 何に対してカジュアルなのか（価格か、空気か、関係性か）。守られている一線はどこか

### 1.4 和

- **本質**: 間・非対称・素材の正直さ・季節・削ぎ落とし。様式ではなく**美意識**である
- **5軸の典型**: 静・伝統寄り。温度と距離は店・企業により大きく変わる
- **表現の一般解**: 余白を「間」として設計、非対称の構図、にじむ・満ちる系の控えめなMotion、和紙・墨・木のような質感、縦組みの検討
- **罠**: 筆文字・赤い落款・桜・波紋の自動適用。これは和ではなく観光ポスターの記号である
- **問い**: このブランドの「和」は何の和か — 素材か、所作か、季節感か、静けさか、もてなしか

### 1.5 モダン

- **本質**: 明晰さ。構造が見えること。装飾ではなく論理が美しさを作る
- **5軸の典型**: 冷寄り・革新寄り。速度は中庸
- **表現の一般解**: 厳密なグリッド、サンセリフ、幾何学形態、切れのよいTransition、機能から導かれた色
- **罠**: 明晰さの追求が無個性に落ちる（どの会社に出しても成立する画面 — QUALITY_STANDARD 1章 不合格例）。モダンこそブランド固有の핵が最も必要なトーン
- **問い**: このブランドが「削って残した一つ」は何か。それが画面の主役になる

### 1.6 北欧

- **本質**: 機能と温かさの同居。厳しい環境が生んだ、光と素材への敬意。簡素だが冷たくない
- **5軸の典型**: 静〜中・軽・温・親密
- **表現の一般解**: 白と自然素材の色、柔らかい光、幾何学と有機形の同居、丁寧で控えめなMotion、実用の中の遊び1点
- **罠**: 「白＋木目＝北欧」の記号化。北欧は色ではなく、機能と温かさを両立させる**態度**である
- **問い**: このブランドにおける「機能」と「温かさ」はそれぞれ何か

### 1.7 未来感

- **本質**: まだ見ぬものへの予感。技術が生む畏怖と期待。「先に行っている」という信頼
- **5軸の典型**: 動・冷・革新
- **表現の一般解**: 暗い地色に制御された発光、精密なデータ的表現、奥行き（Perspective）、鋭い立ち上がりのeasing
- **罠**: 紫グラデーション＋パーティクルは未来感ではなくAI量産出力である（QUALITY_STANDARD 9章 量産感チェックに直撃）。未来「風」の装飾は、示す未来の中身がない証拠
- **問い**: このブランドが実際に見せられる「未来の具体」は何か。それが無いなら未来感トーンを選ばない

### 1.8 ナチュラル

- **本質**: 作為が見えない丁寧さ。素材・土地・生命との接続。あるがままに見せるための膨大な調整
- **5軸の典型**: 静・軽〜中・温・親密
- **表現の一般解**: 生成り・土・草木の色、不均質なテクスチャ、ゆらぎのある有機的Motion、自然光の写真
- **罠**: 「緑を置けばナチュラル」。過度の脱色・淡色化で貧相になる。無添加=無設計ではない
- **問い**: 何の「自然」か — 素材か、製法か、土地か、暮らし方か。写真にその自然が写っているか

# 2. 感情知識 — Emotion Library

**語彙の階層**: 感情（本章）は感情曲線に書く**目的**である。強度語彙（静けさ・期待・インパクト・集中・解放・余韻 — QUALITY_STANDARD 7章）はRhythm Scoreに書く**手段**である。混同しない。「圧倒」という目的を「インパクト」という強度で実現する、という関係にある。

### 感情の文法 — 個別知識より先に守ること

1. **感情は絶対値ではなく変化で生まれる。** 圧倒は静寂の後で最大になる。幸福は不安の解消の後で深くなる。感情を設計するとは、直前の状態を設計することである
2. **遷移には準備が要る。** 感情は瞬間移動しない。高揚から静寂へ移るには解放を挟む。遷移表（下記）に反する接続は、Transitionの唐突さ（QUALITY_STANDARD 6章）として現れる
3. **同じ感情の連続は減衰する。** 2回目の圧倒は半分しか効かない。3回目は疲労になる
4. **感情曲線にはピーク≥1と静けさ≥1が必須**（QUALITY_STANDARD 3章 [Major]）

**主要な遷移の相性**

| 遷移 | 評価 | 備考 |
| --- | --- | --- |
| 静寂 → 圧倒 | 最強 | 落差が最大のとき圧倒は最大 |
| 期待 → 圧倒/感動 | 強 | 吸ってから止める（CDE 7.1） |
| 圧倒 → 静寂/余韻 | 必須 | 受け止めの時間。省略は[Blocker]級（QUALITY_STANDARD 7章） |
| 安心 → 期待 | 良 | 序盤の定石。迎え入れてから誘う |
| 高揚 → 静寂 | 直結は不可 | 解放を挟む |
| 圧倒 → 圧倒 | 原則不可 | 連続≤2の枠内でも減衰を覚悟する |

### 2.1 期待

- **本質**: 「次に何かがある」という予感。情報の意図的な不足＋方向の提示
- **生む条件**: 見切れ、予告（次の影を見せる）、問いかけ、変化の途中を見せる、省略
- **壊す条件**: 最初に全部見せる。期待させて何もない（空振りは信頼を削る）
- **位置**: 冒頭とすべてのピークの直前。呼吸は「吸う」

### 2.2 安心

- **本質**: 予測できること。迎え入れられている感覚。信頼の土壌
- **生む条件**: 一貫したトーン、現在地がわかるナビゲーション、人の顔と言葉、丁寧な速度
- **壊す条件**: 突然の大きな動き・音、現在地の喪失、操作を奪われる感覚（スクロールジャック）
- **位置**: 序盤（世界観への入口）とCTA前（Trust Before Action — 憲章7.7）に必須

### 2.3 圧倒

- **本質**: 自分より大きいものに触れた感覚。畏怖に近い
- **生む条件**: スケールの落差、静寂からの転換、密度の一点爆発、視界いっぱいの単一対象
- **壊す条件**: 準備（吸う区間）なしの唐突な提示。連発
- **規則**: 1ページ1〜2回まで。直前に「吸う」、直後に「吐く」を必ず置く。記憶層（CDE 4.4）の第一候補

### 2.4 静寂

- **本質**: 音が消える感覚。注意が澄み、直前の体験が定着する
- **生む条件**: 大きな余白、動きの完全な停止、短い言葉、低い視覚密度
- **壊す条件**: 常時動き続ける装飾、詰まったレイアウト。静寂を「未実装」と誤解した中途半端な埋め草
- **注意**: 静寂は空白ではなく設計物である（音楽の休符 — CDE 7.5）

### 2.5 余韻

- **本質**: 体験が去った後も内側で響き続ける状態。記憶への橋
- **生む条件**: ピークの要素を小さく反響させる（残像・エコー）、緩やかな減衰、沈黙、短い言葉ひとつ
- **位置**: ピーク直後とページ末尾。**体験の記憶は終わり方で決まる**（ピークエンドの法則 — 本書7.6）ため、末尾の余韻はCTAより先に設計する

### 2.6 高揚

- **本質**: 上へ向かうエネルギー。加速していく感覚
- **生む条件**: テンポの漸増、明度・彩度の上昇、リズミカルな要素の連続、前進感のあるMotion
- **壊す条件**: 長すぎる高揚（疲労に変わる）、理由のない加速
- **注意**: 高揚の後は解放で受ける。急停止させない

### 2.7 幸福

- **本質**: 満ち足りた温かさ。「ここは良い場所だ」という体感
- **生む条件**: 人・食・光の写真の質、柔らかい色温度、余裕のあるテンポ、節度あるユーモア
- **壊す条件**: 作為的なストックフォト的笑顔、過剰な演出（幸福は静かな感情である）

### 2.8 感動

- **本質**: 積み上がった感情の総和が閾値を超えて溢れる状態。**単独では作れない合成感情**
- **生む条件**: 物語の誠実な積層 × 真実の細部（作り話でない具体）× 適切なタイミングのピーク
- **壊す条件**: 感動を直接狙う演出（お涙頂戴）。感動は狙うと逃げる。誠実な積み上げの**結果**としてのみ起きる
- **注意**: 感情曲線に「感動」と書くときは、そこまでの全Sceneが伏線として機能しているかを検査する

# 3. Experience知識 — 他分野からの設計原理

抽出してよいのは**設計原理のみ**。表現様式・素材・コードの模倣は禁止（憲章6章）。研究対象はWEDNES、Apple、Stripe、Active Theory、Locomotive、Awwwards掲載・受賞サイト、ヤマトグループなどの高品質な企業周年・ブランドストーリーサイト等。

### 3.1 映画

- **原理**: ①ショットの長さが感情を作る（長回し=没入、速いカット=緊張）②引きから寄りへ — 状況を見せてから感情に寄る ③モンタージュ — 2つの画の並置が、どちらにもない意味を生む ④カットには動機が要る
- **翻訳**: Sceneの高さと情報密度がショットの長さに相当する。Scene冒頭は引き（全体）、核心で寄り（細部）。画像の並置で語らせ、説明文を減らす
- **採用条件**: Storyが確定している（Story Before Motion）。禁忌: 訪問者の再生ヘッドを奪って映画を再現すること（CDE 4.2）

### 3.2 ゲーム

- **原理**: ①主体性 — 自分で見つけたものは与えられたものより記憶に残る ②透明なチュートリアル — 遊びながら学ばせ、説明で教えない ③環境ストーリーテリング — 背景・小物が物語を語る ④緊張曲線 — ボスの前には準備の間がある
- **翻訳**: 操作説明なしで自然に誘導する。スクロールの先に「発見」を仕込む。装飾ではなく物語を含んだディテールを置く
- **禁忌**: 発見の演出が実用情報（住所・営業時間等）への到達を妨げること（QUALITY_STANDARD 13章）

### 3.3 建築

- **原理**: ①シークエンス — 圧縮（狭い入口）が解放（開けた空間）を大きくする ②光の設計 — 明暗が動線と感情を導く ③素材の正直さ ④プロムナード — 歩く速度で体験が設計されている
- **翻訳**: Heroから最初の展開への圧縮→解放。Scene間の「間」。明度の設計を感情曲線に従わせる
- **採用条件**: ページを「空間」として設計する意図があるとき。Rhythm設計（CDE 7章）の主要な根拠

### 3.4 展示会

- **原理**: ①キュレーション — 何を展示しないかが展示の質を決める ②1室1作品の贅沢 ③順路と自由の均衡 ④キャプションの節度 — 作品より雄弁な解説は失敗
- **翻訳**: 1Scene1目的（QUALITY_STANDARD 5章）。削る勇気（Restraint — 憲章7.9）。コピーは作品（写真・商品）より前に出ない

### 3.5 Apple的プロダクト提示

- **原理**: ①1画面1主張 ②主役の孤立 — 製品以外のすべてが沈黙する ③スペックの体験翻訳 — 数値ではなく「それで何ができるか」を語る
- **禁忌**: 白背景に浮遊する製品＋スクロール連動動画という**表現様式そのものの模倣**。抽出するのは「主役の孤立」「体験翻訳」という原理だけである

### 3.6 Awwwards受賞級サイト

- **原理**: ①冒頭3秒の世界観宣言 — 最初の1画面でこのサイトの時間の流れ方を約束する ②技術は世界観の従属物 — WebGLが目的化した受賞サイトは古びるのが早い ③細部の解像度 — カーソル・hover・端の処理まで世界観が貫通している
- **禁忌**: 「受賞サイト風」演出の記号的引用。それこそが量産感の正体である。参考にするのは完成形ではなく、**なぜその判断に至ったか**の推論

### 3.7 Luxury Hotel

- **原理**: ①アライバルシークエンス — 到着前から体験は始まっている（予約確認の文面、送迎、エントランスまでの導線）②先回りするサービス — 望む前に用意されている。サービスの姿は見えない ③五感の統一 — 香り・音・照度・温度が一つの世界観に揃う
- **翻訳**: 読み込み〜Heroが「到着」の体験になる。ナビゲーション・次の情報が求める半歩先に現れる。ページ全体の質感（色温度・速度・言葉遣い）を統一する
- **採用条件**: もてなし・滞在が価値のブランド（宿・レストラン・サロン等）

### 3.8 Restaurant

- **原理**: ①コースの構成 — 先付から甘味までの順番は感情曲線そのものである（軽く開き、主菜で頂点、甘味で余韻）②一皿ずつ出す — 全部を一度にテーブルに並べない ③皿の余白 — 盛らないことが主役を立てる ④作り手の顔 — 誰が作っているかが信頼になる
- **翻訳**: 情報開示の順番をコースとして設計する。1Sceneに1皿。写真は引き算の構図。人のSceneを信頼形成に置く（QUALITY_STANDARD 18章: CTA前の信頼要素Scene≥2）

### 3.9 Museum

- **原理**: ①照度設計 — 暗さが集中を作り、一点の光が視線を決める ②鑑賞距離 — 作品ごとに正しい距離がある（寄れるもの、引くべきもの）③展示替えの呼吸 — 部屋ごとの密度の緩急
- **翻訳**: 暗い地色のSceneでは光=視線誘導装置として設計する。写真・作品ごとに「見せる距離」（画面内サイズ）を意図して決める。部屋の緩急はRhythm Scoreに対応する

# 4. UI知識 — 視覚設計の原理

本章の数値的下限はQUALITY_STANDARD 10〜12章・14章が正である。本章は「なぜそうするか」と判断基準を与える。

### 4.1 余白

- **本質**: 余白は「余り」ではない。**主役を指定する沈黙**である
- **判断基準**: 要素を置くたびに問う — この余白は何を立てているか。答えられない余白は未設計、答えられない要素は過剰
- **採用条件**: 静・重・高級・和トーンでは余白が主要な表現手段になる。Rhythmの「吐く」区間は余白で作る
- **禁忌**: 余白の圧迫・要素の接触（QUALITY_STANDARD 11章 [Blocker]）。スケール外の場当たりな間隔

### 4.2 Typography

- **本質**: 文字は情報の器であると同時に、**ブランドの声**である。書体・級数・字間が声質・声量・話す速さに相当する
- **判断基準**: ①可読の下限（本文16px以上・行間1.5〜1.9・行長 — QUALITY_STANDARD 10章）は演出と交換しない ②ジャンプ率（見出しと本文のサイズ差）がトーンの音量差 — 静かなブランドはジャンプ率を抑え、動的なブランドは大きく取る ③ファミリー≤3
- **禁忌**: 装飾フォントの本文使用。ライセンス未確認フォント

### 4.3 Contrast

- **本質**: 差が意味を作る。コントラストは「どこを見るべきか」の宣言である
- **判断基準**: ①テキストコントラストはWCAG AA（4.5:1 / 3:1 — QUALITY_STANDARD 14章）が下限 ②演出のために本文コントラストを下げることは禁止 ③強いコントラストは1画面に1箇所へ絞る（全部強い=全部弱い、はMotion強度と同じ原理 — CDE 6.6）

### 4.4 Hierarchy

- **本質**: 視線の順序の設計。訪問者は読む前に「どの順で見るか」を無意識に決めている
- **判断基準**: ①1Sceneの最強調は原則1つ（QUALITY_STANDARD 12章）②意図した視線順と実際の視線順が一致するかをスクリーンショットで検査 ③情報の重要度と視覚的強さの一致
- **禁忌**: 装飾がCTAより目立つこと。見出し階層と視覚階層の食い違い

### 4.5 Grid

- **本質**: 秩序の宣言。グリッドは見えない骨格として「このページは制御されている」という安心を作る
- **判断基準**: ①グリッドを敷いてから、**破る場所を設計する** — 破格は1画面1箇所まで、感情曲線上の理由があるときのみ ②整列の乱れは意図か事故か、必ず区別できる状態にする
- **採用条件**: モダン・未来感トーンではグリッドの厳密さ自体が表現。和・ナチュラルでは非対称グリッドの検討

### 4.6 Color

- **本質**: 色は最速の感情装置であり、最強のブランド記憶装置である
- **判断基準**: ①基調＋補助＋強調1色に絞ることから始める ②ブランド固有色はBrand Document（実物・空間・素材の色）から抽出する。トレンドパレットから選ばない ③彩度・明度は感情曲線に従わせられる（序盤は抑え、ピークで開くなど）
- **禁忌**: フレームワークのデフォルト色・AI出力にありがちな紫グラデーションの放置（QUALITY_STANDARD 9章 量産感チェック）

### 4.7 Composition

- **本質**: 画面内の重さと方向の設計。構図は視線に「入口と出口」を与える
- **判断基準**: ①対称=静・威厳、非対称=動き・緊張。トーンの5軸から出発する ②見切れは期待の装置（本書6.4）③要素の視線（人物の目線・物の向き)は訪問者の視線を運ぶ — 主役へ向ける
- **禁忌**: すべてのSceneが中央揃え1カラムの単調（構図の単調はRhythmの単調と同じ疲労を生む）

# 5. Motion知識 — Motion Technique Library

**共通規則（全技法に適用）**
- 採用には憲章7.4の理由リストへの着地が必須。判定手順はCDE 6章・9.2
- 同型の連続≤2（QUALITY_STANDARD 5章・6章）
- `prefers-reduced-motion`の代替が成立しない技法は使えない
- 実装はtransform / opacity中心（QUALITY_STANDARD 16章）。モバイルは縮小ではなく再設計（同15章）

### 5.1 Parallax

- **知覚効果**: 層の速度差が奥行きの錯覚を生む。世界に「厚み」が生まれる
- **感情効果**: 没入・空間性・静かな豊かさ
- **採用条件**: 層に意味があるとき（前景=いま、主役=主題、背景=世界）。空間性が世界観に寄与するブランド
- **禁忌**: 意味のない多層化。本文テキスト層を動かす。速度差の付けすぎ（酔い）
- **注意**: 効果は控えめなほど品がよい。移動量はビューポートの数%台から検討する

### 5.2 Pin（固定）

- **知覚効果**: スクロールしても対象が留まる。時間が止まり、対象の**変化**に注意が集まる
- **感情効果**: 注視・集中・「見せられている」感覚
- **採用条件**: 1つの対象の変化・段階を順に見せたいとき（製法の工程、空間の遷移など）
- **禁忌**: スクロール入力が演出を進めない時間を作ること — それはスクロールジャックである（QUALITY_STANDARD 13章: 入力無視時間0秒）。内容量に対して長すぎるpin距離
- **注意**: pin区間のスクロール量は「読む・見る」のに必要な時間から逆算する。長いpinはRhythm上「集中」区間として譜面に記載する

### 5.3 Mask

- **知覚効果**: 見える範囲の制御。窓の向こうに世界がある感覚
- **感情効果**: 期待・発見・開示の劇性
- **採用条件**: 情報開示の順番に演出的な理由があるとき。Scene間の受け渡し（前Sceneの終わりが次の窓になる）
- **禁忌**: 読みたい・見たいものを隠し続けること。開示の焦らしすぎ（期待は空振りすると信頼を削る — 本書2.1）

### 5.4 Reveal（出現）

- **知覚効果**: 無から有への遷移。注意の誘導と順序の提示
- **感情効果**: 期待の解決・リズムの刻み
- **採用条件**: 出現の順序に意味があるとき（読む順・重要度順）
- **禁忌**: **全要素が同型のRevealで出現すること**（QUALITY_STANDARD 9章 量産感チェック第1項）。視界外で終わっている空回りReveal
- **注意**: Revealは最も安易に使われる技法である。理由を書けないRevealは静的表示に置き換える

### 5.5 Scale

- **知覚効果**: 大きさの変化はカメラの寄り引きとして知覚される
- **感情効果**: 拡大=接近・親密・圧倒への助走。縮小=引き・俯瞰・相対化
- **採用条件**: 距離の変化に物語上の意味があるとき（細部へ寄る、全体を見渡す）
- **禁忌**: レイアウトを動かすscale（CLS violation）。本文テキストのscale
- **注意**: transformで実装し、周囲のレイアウトに影響させない

### 5.6 Rotate

- **知覚効果**: 回転は不安定さ・機構・遊びを感じさせる
- **感情効果**: 軽さ・ユーモア・機械的精密さ（文脈による）
- **採用条件**: 軽・動トーン、または機構・工程の説明に回転が実在するとき
- **禁忌**: 静・重・高級トーンでの装飾回転。大面積の回転（酔い）。連続回転し続ける装飾

### 5.7 Perspective / 3D

- **知覚効果**: 三次元の奥行き。画面が「面」から「空間」になる
- **感情効果**: 未来感・没入・技術的信頼
- **採用条件**: 空間性がブランド価値に直結するとき。DOM/CSS/GSAPで代替しにくい体験価値があるときのみThree.js/WebGLを検討（憲章8章 技術方針）
- **禁忌**: 目的化した3D。低性能端末での破綻（QUALITY_STANDARD 16章はWebGL採用時も免除されない）
- **注意**: コストが最も高い技法。採用時は個別のJS予算を記録する

### 5.8 Camera（視点の移動）

- **知覚効果**: ページ全体を1つの空間とみなし、スクロールを視点移動として統一するメタファ
- **感情効果**: 一貫した世界の中を旅している感覚。強い没入
- **採用条件**: ページが最初から「空間」として設計されているときのみ。視点移動の論理（どこからどこへ、なぜ）が全Sceneで一貫していること
- **禁忌**: Sceneごとに視点の論理が変わること（世界が壊れる）。部分的な思いつきでのカメラ演出
- **注意**: 採用するならStory・Scene設計の段階（B3〜B4）で決める。Motion設計段階（C1）での後付けは不可能である

### 5.9 Transition（Scene間の受け渡し）

- **類型**: 溶ける（dissolve）／受け渡す（前Sceneの要素が次Sceneの一部になる）／切る（cut）／変形（morph）
- **トーン対応**: 静・重=溶ける・受け渡す。動・軽=切る・跳ぶ。精密・未来=morph
- **採用条件**: 前Sceneの終わりと次Sceneの始まりの関係（継続か、転換か）をStoryで説明できること
- **禁忌**: 全境界が同型（同一Transition連続≤2 — QUALITY_STANDARD 6章）。Transition中のスクロールロック（1秒超は[Blocker]）。切れ目の白紙・崩れ

# 6. Scroll知識 — スクロール体験の技法

Rhythmの理論体系はCDE 7章（呼吸モデル・6強度語彙・譜面・規則）が正である。本章はそれを実現する技法カタログを与える。

### 6.1 Rhythm（参照）

理論はCDE 7章。本章の技法はすべて譜面（Rhythm Score）の上に配置されて初めて機能する。譜面なしに技法を置くことは、楽譜なしに音符を撒くことである。

### 6.2 Scene

- **本質**: 体験の最小単位。1目的（伝える情報＋起こす感情）を持つ（QUALITY_STANDARD 5章）
- **判断基準**: ①Sceneの分割は「感情が変わる場所」で切る（情報の種類ではなく）②1Sceneに5つ以上のメッセージがあれば分割か削除 ③目的が重複するSceneは統合
- **Sceneの高さ**: 担当する呼吸で決める。「吸う」Sceneは短く、「集中」Sceneは内容量に比例、「吐く」Sceneは内容より長く

### 6.3 Tempo

- **本質**: スクロール距離あたりの情報量・変化量。訪問者が体感する「ページの速度」
- **判断基準**: ①読む区間（集中）は密度を上げ、変化を減らす ②感じる区間（期待・余韻）は密度を下げ、スクロール距離に余裕を持たせる ③テンポの変化そのものが緩急の装置 — 一定テンポのページは内容が良くても疲れる（Scroll Rhythm — 憲章7.6）
- **禁忌**: 内容量と釣り合わないスクロール距離（空疎な水増し、または詰め込み圧迫）

### 6.4 Curiosity — 好奇心の技法カタログ

各Sceneは「次を見たい」を作る（憲章7.5）。ただし**手法は毎回変える**（同一フック連続≤2 — QUALITY_STANDARD 5章）。

| 技法 | 仕組み | 使用条件 |
| --- | --- | --- |
| 見切れ | 次の要素の一部が画面端に見えている | 視覚的な連続性があるSceneの間 |
| 予告 | 次のSceneの影・断片・言葉を先に出す | Storyに伏線構造があるとき |
| 未完の形 | 図形・線・文章が完結せず次へ続く | 抽象表現が世界観に合うとき |
| 問いかけ | コピーが問いで終わる | 答えが次のSceneに実在するときのみ |
| 視線の先 | 写真内の人物・物の視線がスクロール方向を向く | 人・物の写真があるScene |
| 省略 | あえて語らないことで想像させる | ブランドに語らない品格があるとき |
| 変化の途中 | 何かが変わり始めた状態でSceneが終わる | 次Sceneがその変化の完了を受けるとき |

### 6.5 Surprise

- **本質**: 予測を一度作らせてから、世界観の内側で裏切る
- **判断基準**: ①予測の形成が先（パターンを2回見せてから3回目で外す、など）②頻度は1ページ1〜2回 — 驚きの連発は驚きを殺す ③裏切りは世界観の**内側**で行う。世界観の外への逸脱は驚きではなく断絶（QUALITY_STANDARD 2章）
- **禁忌**: 驚かせること自体が目的の演出（Motion Has Meaningの理由リストに「驚かせる」は無い — 期待を生む・感情を変える文脈でのみ成立する）

### 6.6 Pause

- **本質**: 訪問者が止まる場所の設計。スクロールは連続ではなく「進んでは止まる」の反復である
- **判断基準**: ①止まってほしい場所（ピーク・核心のコピー・記憶層）に、止まりたくなる密度と構図を置く ②**どの停止位置でも画面が破綻しない**（再生ヘッドは訪問者のもの — CDE 4.2）③意図しない停止（読み込み・カクつき）はPauseではなく事故である

### 6.7 Reveal（開示戦略）

- **本質**: Motion技法のReveal（5.4）の上位概念。ページ全体で「何を・いつ・どの順で見せるか」の戦略
- **判断基準**: ①開示順はStoryの開示順理由（QUALITY_STANDARD 4章）に従う ②最初に全部見せない（ゲームの発見原理 — 本書3.2）。ただし実用情報への最短経路は常に確保する ③最後まで取っておく1枚（記憶層候補）を決めてから、途中の開示を逆算する

# 7. UX知識 — 訪問者の心理と行動

### 7.1 認知負荷

- **本質**: 訪問者が一度に処理できる情報量には上限がある。超えた分は「読まれない」のではなく「疲労として蓄積される」
- **判断基準**: ①1Scene1メッセージ（QUALITY_STANDARD 5章）②同時に動く要素を絞る（Motionの同時性 — CDE 6.6）③訪問者がいま「読むモード」か「感じるモード」かを区別し、読むモードの区間では演出を弱める（CDE 6.4）
- **禁忌**: 読解中のテキスト周辺の常時アニメーション。選択肢・情報の並列提示過多

### 7.2 視線誘導

- **本質**: 視線は設計できる。誘導装置には強さの序列がある
- **装置の序列（強い順）**: 動き > 人の顔・視線 > コントラスト > サイズ > 色 > 位置
- **判断基準**: ①1画面で複数の装置を競合させない（最強調1つ — QUALITY_STANDARD 12章）②最強の装置「動き」を使うことは、他のすべてから注意を奪うことである — Motionの採用判断（CDE 6章）にこのコストを含める ③写真内の視線は主役・スクロール方向へ向ける（本書6.4）

### 7.3 CTA

- **本質**: CTAは物語の結末であり、信頼の出口である。ボタンではなく**状態**を設計する — 「押したくなっている状態」を作るのはCTAより前の全Sceneである
- **判断基準**: ①CTA前に信頼要素Scene≥2（QUALITY_STANDARD 18章）②表現はブランドトーンに従う（唐突な販促バナー調は断絶）③迷いの除去 — 押した後に何が起きるか（返信までの時間、予約の流れ）を先に伝える
- **禁忌**: Hero直下の唐突なCTA。閲覧を妨げる強制ポップアップ・過剰な追従（同章 [Major]）

### 7.4 Trust — 信頼形成

- **本質**: 信頼は主張ではなく証拠から生まれる。「こだわっています」は信頼を作らない。こだわりの**具体**が作る
- **信頼の4源泉**: ①具体性（数値・固有名詞・工程の実際）②人（顔・言葉・手)③実績・第三者の声 ④一貫性（ページ全体でトーン・事実が揺れない）
- **判断基準**: Brand Documentの「信頼の源泉」軸（CDE 3.2）で、このブランドの信頼が4源泉のどれに立つかを特定してからSceneを設計する
- **禁忌**: 誇張・虚偽・確認していない事実の記載（QUALITY_STANDARD 17章: 事実情報はクライアント確認済みが[Blocker]）

### 7.5 Decision — 決断の心理

- **本質**: 訪問者が行動しない理由は「魅力不足」より「不安・面倒・忘却」であることが多い
- **判断基準**: ①不安の除去（何が起きるかの予告、キャンセル・変更の条件）②面倒の除去（入力項目の最小化、電話・地図への1タップ到達）③忘却への手当て（体験の記憶層がブランド名・場所と結びついているか）
- **禁忌**: 不安を煽って決断させること（下記7.7）

### 7.6 記憶の心理学 — ピークエンドの法則

- **本質**: 体験の記憶は、平均ではなく**ピークと終わり方**で決まる
- **Bloom-Lでの使い方**: これがCDE 4.4「記憶層は1〜2箇所に絞る」とCDE 7.4「末尾は余韻と行動」の心理学的根拠である。全Sceneを均等に磨くより、ピーク1つと末尾を磨く方が記憶に残る
- **併せて使う原理**: 単純接触（一貫したモチーフの変奏が親しみを作る — 反復ではなく変奏）、返報性（先に価値を与える — 出し惜しみしない情報開示が信頼を作る）、社会的証明（第三者の声。ただし本物のみ）

### 7.7 心理学の倫理 — この節が7章全体を拘束する

心理学の知識は、**訪問者の利益になる形でのみ**使う。

- 使ってよい: 理解を助ける、不安を取り除く、判断材料を揃える、記憶に残す
- 禁止: 恐怖・焦り・羞恥による行動喚起、偽の希少性（残りわずか演出）、解除しにくいUI、注意の搾取（CDE 10章-10 ダークパターンの禁止）

判定基準: **その手法を、訪問者本人に説明できるか。** 説明したら怒られる手法は使わない。

| 緩急がつかない | CDE 7章（譜面）→ 6.3（Tempo）→ 4.1（余白） | 譜面が先。技法は譜面の後 |
| このMotionを使うべきか | CDE 6章（判断基準）→ 5章（技法の禁忌） | 理由→技法の順。技法から引かない |
| 画面が美しくない | 4章（UI）— 階層→余白→タイポ→色の順で検査 | 色から直さない。原因は大抵階層か余白 |
| CTAが弱い・唐突 | 7.3〜7.5（CTA・Trust・Decision） | ボタンではなく前のSceneを疑う |
| どこか量産感がある | 1章の各「罠」＋ QUALITY_STANDARD 9章 | 罠への該当を総当たりで検査 |
| 知識同士が矛盾する | 8.3 | |
| 本書に該当知識がない | 8.4 | |

### 8.2 共通採用手順 — どの知識も、この5段階を通す

```
Step 1: 状況の特定
        いまの判断はCDEフローのどの段階か。何を決めようとしているか

Step 2: 候補の列挙
        該当章から候補知識を2つ以上引く（1つしか引かないと、それが答えに見える）

Step 3: 条件照合
        各候補の「採用条件」を満たすか。「禁忌」に該当しないか
        禁忌該当 → 即棄却（掟3）

Step 4: whyChain検査（CDE 9.5）
        その知識を使う理由の連鎖が、Brand Document・感情曲線・
        訪問者理解のいずれかに着地するか
        着地しない → 棄却。知識の引用は理由にならない（掟2）

Step 5: 上書き確認と記録
        Brand Documentの固有記述と突き合わせ、一般解が固有記述に
        反すれば固有記述に従う（掟1）
        採用したら [表現判断]/[根拠] 形式で記録（CDE 3.4）
```

### 8.3 知識同士が矛盾したとき

1. まず、矛盾が本物か確かめる。多くの場合、採用条件の読み落としである（例: Surpriseと安心の矛盾 → Surpriseは「世界観の内側で」が条件であり、安心の土台と両立する）
2. 本物の矛盾なら、憲章12章の意思決定原則を上から順に適用する（下限 > 訪問者の目的 > ブランド適合 > 一般的な良さ > 減らす）
3. 本書と上位文書（憲章・QUALITY_STANDARD・CDE）の矛盾は、判断せず停止して報告する

### 8.4 本書に知識がないとき

1. 最も近い原理を持つエントリから推論する（例: 「工房見学」体験 → 3.9 Museum＋3.8 Restaurantの原理を合成）
2. 推論した判断には`[推定]`ラベルを付け、根拠を記録する（CDE 9.4）
3. 実案件で検証されたら、本書への追記を提案する。**検証前の思いつきを本書に書き足すことは禁止**（CDE 付記・改訂ルールと同じ運用）

### 8.5 知識の悪用 — 検出パターン

以下の使い方を検出したら、その判断を破棄して該当段階へ戻る。

| 悪用パターン | 症状 | 戻り先 |
| --- | --- | --- |
| 辞書引きデザイン | トーン名→一般解を直行適用し、上書き（5.4）が一度も起きていない | CDE A3（ブランド理解） |
| 技法起点 | 5章の技法から発想し、Sceneが技法のために作られている | CDE B4（Story Before Motion違反） |
| 知識の権威化 | 「本書に書いてあるから」が根拠欄に書かれている | whyChainのやり直し |
| カタログ消化 | 6.4の好奇心技法を全種使うこと自体が目的化している | Rhythm譜面へ（技法数は譜面が決める） |
| 心理学の搾取的利用 | 7章の知識が訪問者の不利益方向に使われている | 即時削除（7.7。戻り先はない） |
